Anthropic 团队如何使用 Claude Code

不同团队在日常工作中使用 Claude Code 的實戰經驗與最佳實踐

文件版本 2.0 來源：Anthropic 官方文件

概述

本文件記錄 Anthropic 不同团队在日常工作中如何使用 Claude Code，包括主要使用情境、工作流程、团队影响及最佳技巧。

Claude Code 用於产品开发

团队概述

产品开发团队負責建構 Claude 及其界面。他們广泛使用 Claude Code 進行日常开发任務。

主要使用情境

功能文件工作流程

团队建立包含逐步指令的功能文件，定義新功能的预期行为。Claude Code 透過测试驱动开发 (TDD) 协助建立和解讀這些文件，先撰写预期行为，然後 Claude 持續执行直到滿足所有标准。

端對端测试

Claude Code 撰写使用者故事並自動为複雜功能建立端對端测试案例，捕捉人工可能遗漏的边缘案例。团队將测试視为验证代码正確性的主要產出，可更快识别並修正问题。

全端开发

处理面向客戶的變更時，Claude Code 生成新的 React 元件、更新数据库结构並修改 API 路由。团队提供明確定義的子任務，讓 Claude 跨全端实现。

代码审查协助

Claude Code 协助识别边缘案例和重構机会，指出潛在陷阱或建議改進，使代码更健壯且易於维护。

团队影响

  • 加速开发周期：過去需要數週的功能，現在透過 Claude Code 处理重複性实现工作可更快完成。
  • 透過测试提升代码品質：使用 Claude Code 的测试驱动开发可捕捉自身错误，尤其是請 Claude 先生成测试再撰写代码時特別有效。

产品开发团队最佳技巧

培養任務分類直覺

学习區分適合非同步处理的任務（周邊功能、原型製作）與需要同步监督的任務（核心业务逻辑、关键修復）。产品边缘的抽象任務可使用「自動接受模式」处理，核心功能則需更密切监督。

撰写清晰详细的提示

當元件名稱或功能相似時，請求時務必极度具體。提示越详细，越能信任 Claude 独立工作，避免對代码庫错误部分進行意外變更。

Claude Code 用於安全工程

团队概述

安全工程团队专注於保護软件开发生命周期、供應鏈安全及开发环境安全。他們广泛使用 Claude Code 撰写和调试代码。

主要使用情境

複雜基礎設施调试

处理事件時，他們將堆栈追踪和文件提供給 Claude Code，要求追踪代码庫中的控制流程。這顯著缩短生產问题的解決時間，原本需要 10-15 分鐘手動扫描代码的问题，現在約 5 分鐘即可理解。

Terraform 代码审查與分析

對於需要安全批准的基礎設施變更，他們將 Terraform 计划複製到 Claude Code 中询问「這會做什麼？我會后悔嗎？」這创造更紧密的回饋循環，讓安全团队更容易快速审查和批准基礎設施變更，减少开发流程中的瓶颈。

文件综合與运行手册

他們讓 Claude Code 攝取多個文件來源，建立 markdown 运行手册、故障排除指南和概述。他們使用這些精簡文件作为调试實際问题的背景，创造比搜索完整知識庫更有效率的工作流程。

测试驱动开发工作流程

他們現在不再使用之前的「設計文件 → 临时代码 → 重構 → 放棄测试」模式，而是向 Claude Code 請求偽碼，引导其進行测试驱动开发，並定期检视以在卡住時引导方向，生成更可靠且可测试的代码。

情境切換與项目新手上路

贡献現有项目（如「dependant」安全批准工作流程網頁应用）時，他們使用 Claude Code 撰写、审查和执行以 markdown 撰写並存放在代码庫中的规格，使有意義的贡献從數週缩短至數天。

团队影响

  • 缩短事件解決時間：原本需要 10-15 分鐘手動扫描代码的基礎設施调试，現在約需 5 分鐘。
  • 改善安全审查周期：安全批准的 Terraform 代码审查速度大幅加快，消除开发人員等待安全团队批准時的阻塞。
  • 强化跨功能贡献：团队成員可在數天內（而非數週的背景建立）有意義地贡献项目。
  • 更佳文件工作流程：從多個來源综合的故障排除指南和运行手册创造更有效率的调试流程。

安全工程团队最佳技巧

广泛使用自訂斜線指令

安全工程使用整個 monorepo 中 50% 的自訂斜線指令实现。這些自訂指令簡化特定工作流程並加速重複任務。

讓 Claude 先行動

不再针对代码片段提出针对性问题，他們現在告訴 Claude Code「邊做邊提交你的工作」，讓它自主工作並定期检视，生成更全面的解決方案。

善用於文件撰写

除了程式設計外，Claude Code 擅長综合文件和建立结构化输出。他們提供寫作範例和格式偏好，獲得可立即在 Slack、Google Docs 和其他工具中使用的文件，避免界面切換疲勞。

Claude Code 用於推論

团队概述

推論团队管理存储資訊的内存系統，在 Claude 讀取提示並生成回應時運作。团队成員，尤其是機器学习新手，可广泛使用 Claude Code 弥补知識差距並加速工作。

主要使用情境

代码庫理解與新手上路

团队高度依賴 Claude Code 在加入複雜代码庫時快速理解架構。不再手動搜索 GitHub 仓库，他們询问 Claude 找出哪些文件呼叫特定功能，在数秒內獲得結果，而非询问同事或手動搜索。

包含边缘案例的單元测试生成

撰写核心功能後，他們請 Claude 撰写完整的單元测试。Claude 自動包含遗漏的边缘案例，原本需要大量心力的工作在數分鐘內完成，如同一個可审查的程式設計助手。

機器学习概念解釋

沒有機器学习背景的团队成員依賴 Claude 解釋模型特定功能和設定。原本需要一小時 Google 搜索和閱讀文件的工作現在只需 10-20 分鐘，研究時間减少 80%。

跨语言代码轉換

在不同程式语言中测试功能時，他們解釋要测试的内容，Claude 以所需语言（如 Rust）撰写逻辑，無需僅为测试目的而学习新语言。

指令回想與 Kubernetes 管理

不再内存複雜的 Kubernetes 指令，他們询问 Claude 正確語法，如「如何取得所有 pod 或部署狀態」，並獲得基礎設施工作所需的確切指令。

团队影响

  • 加速機器学习概念学习：研究時間减少 80% — 原本需要一小時 Google 搜索的工作現在只需 10-20 分鐘。
  • 更快的代码庫导航：可在数秒內找到相關文件並理解系統架構，而非询问同事。
  • 完整测试涵蓋：Claude 自動生成包含边缘案例的單元测试，減輕心理負擔同時維持代码品質。
  • 消除语言障礙：可在不熟悉的语言如 Rust 中实现功能，無需学习該语言。

推論团队最佳技巧

先测试知識庫功能

嘗試提問各種问题，看 Claude 能否比 Google 搜索更快回答。如果更快且更準確，它就是工作流程中有價值的节省時間工具。

從代码生成開始

給 Claude 具體指示並請它撰写逻辑，然後验证正確性。這有助於在用於更複雜任務前建立對工具能力的信任。

用於撰写测试

讓 Claude 撰写單元测试可顯著減輕日常开发工作的壓力。善用此功能維持代码品質，無需花時間手動思考所有测试案例。

Claude Code 用於微調

团队概述

微調团队透過配對的輸入输出範例训练模型，使其在特定任務上表現更好。

主要使用情境

管理实验追踪

他們使用 Claude Code 檢查训练运行、迭代代码、管理多個实验的大量日誌。团队处理多個活躍运行，Claude Code 可透過讀取日誌文件快速导航训练仪表板，按需识别特定运行特徵或错误。

建構实验工具

团队成員需要專門的实验工具（如手動审查範例的「氛围检查」界面）時，Claude Code 可在單一工作階段中协助建構這些工具。非开发人員可为特定工作流程建立自己的脚本，存取過去需要工程支援的能力。

代码审查與调试

Claude Code 协助捕捉数据集准备脚本和实验配置中的问题。团队使用它调试分布式训练挑战，並在開始長時間训练作業前验证数据管道正確運作。

指標分析與可视化

他們使用 Claude Code 生成效能指標並建立比較圖表。對於研究報告，他們可快速生成顯示模型效能差異的条形图，無需深入了解資料可视化库即可快速將效能可视化。

团队影响

  • 更快的实验迭代：可在單一工作階段中建構自訂工具，而非等待工程支援。
  • 缩短调试時間：在昂貴的训练运行前捕捉数据管道问题。
  • 工具民主化：非开发人員可建立自己的工作流程和分析脚本。
  • 加速報告准备：快速生成效能可视化和指標。

微調团队最佳技巧

建立特定实验工作區

为不同实验保持独立的 Claude Code 工作階段，允許特定背景的协助，避免项目間混淆。

長時間运行前验证

在開始多小時训练运行前，務必使用 Claude Code 验证数据管道和配置，避免代價高昂的错误。

用於临时分析

需要從实验日誌或指標快速獲得見解時，描述您尋找的内容，讓 Claude Code 解析並摘要相關資訊。

Claude Code 用於强化学习工程

团队概述

强化学习 (RL) 工程团队致力於 RLHF（人類回饋强化学习）及相關训练系統。

主要使用情境

狀態检视工具

他們使用 Claude Code 追踪训练作業中的问题。面對包含模型狀態的日誌時，Claude Code 可快速解析並识别问题，减少手動檢查训练配置和输出的時間。

多程序协调

RL 系統需要跨多個程序协调。Claude Code 协助管理和调试分布式系統，撰写脚本监控程序健康並识别协调失敗。

指標管線开发

他們使用 Claude Code 为实验建構指標管線，自動化提取和可视化训练信號如獎勵曲線和損失值。

测试框架开发

Claude Code 协助为 RL 系統元件撰写测试框架，確保在生產部署前验证训练稳定性和预期行为。

团队影响

  • 更快的调试周期：複雜分布式系統问题识别更快。
  • 自動化监控：追踪训练健康的脚本無需手動监督。
  • 改善测试涵蓋：更全面地测试 RL 元件。
  • 簡化指標：自動化提取和可视化训练信號。

强化学习工程团队最佳技巧

针对特定模式自訂 Claude.md 文件

在 Claude.md 文件中新增指示，防止 Claude 重複工具呼叫错误，例如告訴它「执行 pytest 不要 run，不要不必要地 cd — 直接使用正確路徑」。這顯著提升一致性。

使用檢查點密集工作流程

隨著 Claude 進行變更定期提交工作，以便在实验失敗時輕鬆回滾。這使开发採用更具实验性的方法而無風險。

先嘗試一次性，再协作

給 Claude 一個快速提示，讓它先嘗試完整实现。如果成功（約三分之一的機率），您节省了大量時間。如果不成功，再切換到更协作、引导式的方法。

Claude Code 用於法務

团队概述

法務团队透過实验發現 Claude Code 的潜力，以及学习 Anthropic 产品提供的渴望。此外，一位团队成員有個人使用情境，與为家人建立无障碍工具及展示該技術對非开发人員力量的工作原型相關。

主要使用情境

为家庭成員建立自訂无障碍解決方案

团队成員为因醫療诊断而有说话困難的家庭成員建構溝通助手。僅一小時內，他們使用原生语音轉文字建立一個預測文字应用，建議回應並使用语音庫朗读，解決语言治療師推薦的現有无障碍工具中的缺口。

法務部門工作流程自動化

他們建立原型「電話樹」系統协助团队成員联系 Anthropic 的适当律師，展示法務部門如何無需傳統开发资源即可为常見任務建構自訂工具。

团队协调工具

经理建構了 G Suite 应用，自動化每週团队更新並追踪跨产品的法律审查狀態，律師可透過簡單按鈕點擊快速標記需要审查的項目，而非管理电子表格。

快速原型验证解決方案

他們使用 Claude Code 快速建構可向領域专家展示的功能原型（如向 UCSF 专家展示无障碍工具），在投入更多時間前验证想法並识别現有解決方案。

工作风格與影响

  • 在 Claude.ai 规划，在 Claude Code 建構：他們採用兩步驟流程，先在 Claude.ai 头脑风暴和规划，再移至 Claude Code 实现。
  • 视觉優先方法：他們經常使用截图向 Claude Code 展示界面的期望外觀，然後根據视觉回饋迭代。
  • 原型驱动创新：他們強調克服分享「傻」或「玩具」原型的恐懼，因为這些展示可激勵他人看到他們未曾考慮的可能性。

法務部門最佳技巧

先在 Claude.ai 充分规划

使用 Claude 的對話界面在移至 Claude Code 前充分發展整個想法。然後請 Claude 將所有内容摘要为逐步实现提示。

漸進且可视化地工作

請 Claude Code 放慢速度一次实现一個步驟，以便可以複製貼上而不會不知所措。大量使用截图展示界面的期望外觀。

儘管不完美也分享原型

克服隱藏「玩具」项目或未完成工作的衝動 — 分享原型幫助他人看到可能性，並激發通常不互動的部門之間的创新。
